title Conference Registration & Management System - Class Diagram

' --- Core Domain Classes ---

class Registration {
    - registrationNumber: String
    - date: Date
    - attendeeName: String
    - attendeeEmail: String
    - attendeeAddress: String
    - eventOptions: List<String>
    + createRegistration(): Boolean
    + cancelRegistration(): Void
}

class Payment {
    - paymentId: String
    - date: Date
    - purpose: String
    - paymentAmount: Double
    - isSettled: Boolean
    + processPayment(): Boolean
}

class Ticket {
    - ticketNumber: String
    - eventDetails: String
    - isValid: Boolean
    + issueTicket(): Void
    + invalidateTicket(): Void
}

class AttendanceTracker {
    - attendanceId: String
    - sessionDetails: String
    - checkInTimestamp: DateTime
    + recordCheckIn(): Void
    + getSessionAttendanceSummary(): String
}

' --- Relationships & Multiplicities ---

note top of Registration
  <b>Event Options:</b>
  Allows attendees to sign up for specific 
  tracks, workshops, or VIP dinners 
  (e.g., ["Keynote", "Workshop A"]).
end note

note bottom of Ticket
  <b>Ticket Issuance:</b>
  Tightly bound to Registration. 
  A ticket cannot exist without a valid, 
  fully processed registration.
end note
